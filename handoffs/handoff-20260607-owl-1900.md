# Handoff 20260607 owl — Button responsiveness investigation

- Duration: ~45 minutes
- Message Count: ~8
- Compaction Count: 0

## Context

User reported "buttons aren't responding" on the right sidebar after Phase 7 implementation. Previous handoff (`handoff-20260607-owl-1800.md`) described fixing stale `$island*` DOM references, but buttons still appeared non-functional.

## Investigation

Thorough code review: all 391 tests pass, all JS syntax valid, HTML IDs match DOM references, event listener wiring structurally correct. **No single smoking gun found** — the root cause was architectural fragility.

### Root cause

`app.js` `init()` had **zero error boundaries**. If any single line crashed (missing module, null element, IPC failure), ALL subsequent `addEventListener` calls silently failed — no errors visible in console because Electron swallows renderer crashes in the console unless DevTools is open.

### Secondary issue found

**`#right-sidebar-toggle` had no click handler in app.js.** The handler existed only in `right-sidebar.js`. When I removed the right-sidebar.js handler to fix a double-toggle conflict, the button became completely dead. App.js only handled `#sidebar-toggle` (LEFT sidebar).

## Fixes applied

### 1. Error boundaries on init() — `src/renderer/js/app.js`

Every init section wrapped in independent `try/catch` with `console.warn`:
- `window:devtools` IPC handler + preload bridge + F12 key binding
- Right sidebar buttons: `?.` on all `addEventListener` calls
- Each module init (EditorEngine, Inspector, etc.) independently caught
- `$shellHint` null guard added

### 2. Missing right sidebar toggle handler — `src/renderer/js/app.js`

Added `#right-sidebar-toggle` click handler calling `window.RightSidebar?.toggleSidebar()`.

### 3. Removed duplicate toggle listener — `src/renderer/js/right-sidebar.js`

Removed `$toggle.addEventListener('click', _toggleSidebar)` to avoid double-toggle conflict with app.js. The `toggleSidebar` function remains on `window.RightSidebar` API.

### 4. DevTools access — `src/main/ipc-handlers.js` + `src/preload/preload.js` + `src/renderer/js/app.js`

Full IPC chain: F12 → `electronAPI.window.devtools()` → `ipcRenderer.send('window:devtools')` → `win.webContents.toggleDevTools()`.

### 5. Positive logging for button attachment

Replaced blind `?.addEventListener` with a loop that logs `[init] rsidebar buttons attached: X/8` and `[init] MISSING button: <name>` for any null elements.

### Possible remaining issues if buttons still don't work

1. **Caching**: Electron may be serving cached renderer files. Try `rm -rf node_modules/.cache` or `--dev` mode.
2. **Script loading order**: If any earlier IIFE (modes, trust, editor, etc.) throws during its self-execution (not during init), it would prevent later scripts from loading. Check Elements panel for all `<script>` tags.
3. **CSP blocking**: `script-src 'self'` should allow local scripts, but verify in Network tab that all `.js` files load (status 200).
4. **CSS `pointer-events`**: Some overlay might be intercepting clicks. Check with DevTools > Elements > pointer-events audit.
