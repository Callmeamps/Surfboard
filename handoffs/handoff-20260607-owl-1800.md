## Context

Found and fixed a runtime bug in Phase 7: stale references to old `#island` DOM elements that were removed from HTML but still referenced in `app.js`.

### Root cause

When the old floating `#island` was replaced with the right sidebar vertical buttons, the DOM elements (`#island-ai`, `#island-shell`, `#island-extensions`, `#island-bookmark`) were removed from `index.html`. But `app.js` still had:

1. `const $islandBookmark = document.getElementById('island-bookmark')` → `null`
2. `$islandAi.classList.toggle(...)` in `_setShellMode()` → **runtime crash**
3. `$islandShell.classList.remove(...)` in `_hideSidecar()` → **runtime crash**
4. `$islandExt.classList.toggle(...)` in `_toggleExt()` → **runtime crash**

These would throw `Cannot read properties of null` whenever the sidecar was shown/hidden/toggled or the extensions panel was opened.

### Fix applied

- Removed `$islandBookmark` DOM ref (unused — bookmark now handled by `$rsidebarBookmark`)
- Replaced `$islandAi` → `$rsidebarAi`, `$islandShell` → `$rsidebarShell`, `$islandExt` → `$rsidebarExt` with optional chaining (`?.`)
- Right sidebar now starts **visible** (48px) instead of `collapsed` (0px) — matches the design intent
