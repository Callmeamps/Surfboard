# Handoff 20260607 owl — Button responsiveness fix

- Duration: ~30 minutes
- Compaction Count: 0

## Context

User reported "buttons aren't responding" after Phase 7 right sidebar implementation. The stale DOM ref fix (commit `5072111`) resolved crashes from `$island*` → `$rsidebar*` references, but buttons still appeared non-functional.

## Root cause analysis

Thorough code review found **no single smoking gun** — all 391 tests pass, all JS files have valid syntax, HTML IDs match DOM references, and event listener wiring is structurally correct. However, the **lack of error boundaries** in `app.js` init() meant any runtime failure (even from an unrelated module) would silently prevent ALL subsequent event listeners from being attached.

### Issues found and fixed

| Issue | Impact | Fix |
|-------|--------|-----|
| `init()` had zero try/catch blocks | One crash kills ALL button listeners | Added try/catch per section with `console.warn` |
| `$rsidebar*` button listeners used bare `.addEventListener()` (no `?.`) | Would crash init() if element missing | Added `?.` to all rsidebar addEventListener calls |
| `$rsidebarOmnibar.addEventListener` / `$rsidebarExt.addEventListener` had no `?.` | Would crash init() if element missing | Added `?.` |
| Sidecar/chat/addr input listeners had no `?.` | Would crash init() if element missing | Added `?.` and `try/catch` blocks |
| Duplicate `#right-sidebar-toggle` listeners (right-sidebar.js AND app.js) | Both fire on click → double-toggle = net no-op for class | Removed listener from right-sidebar.js, kept app.js handler |
| `$shellHint.textContent = ...` without null check | Would crash if shell-hint element missing | Changed to `$shellHint?.textContent = ...` |
| Mode changes had no visible feedback | User can't tell if mode button click worked | Added `_showModeIndicator()` toast on mode change |

### What the changes accomplish

1. **Graceful degradation**: Each init() section is independently try/caught. If one module fails to load, ALL OTHER button listeners still attach.
2. **Defensive DOM access**: Every `addEventListener` call uses `?.` — no more crashes from missing elements.
3. **Visible mode feedback**: Clicking Edit/Inspect/Actions/Data/Workflows now shows a brief toast "Mode: edit" etc.
4. **No more double-toggle**: Sidebar toggle button has exactly ONE handler (in app.js).

### Files changed

- `src/renderer/js/app.js` — Added try/catch per init section, `?.` on all listeners, mode toast
- `src/renderer/js/right-sidebar.js` — Removed duplicate `#right-sidebar-toggle` listener

### Test results

**391 tests passing across 22 suites** — no regressions.

## Debugging note

If buttons still don't work after these changes, check the Electron DevTools console for `[init]` warnings. Each section logs its failures:
- `[init] RIGHT SIDEBAR BUTTONS FAILED:` — the critical one
- `[init] mode/trust:` — ModeManager/TrustManager didn't load
- `[init] EditorEngine:` — editor module didn't load
