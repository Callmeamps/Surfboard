## Context

### P0 Extensions Bug — Resolved
Root cause: `session.loadExtension` deprecated in Electron 42, silently fails with partitioned sessions.

**Three commits landed:**
1. `afbd875` — Migrated to `ses.extensions.loadExtension()` / `ses.extensions.removeExtension()` API
2. `6a69301` — Added programmatic content script injection + extension API polyfills

### Key findings from testing
- **uBlock Origin**: Background page loads, initializes (`readyToFilter: true`, 10 filter lists). Content scripts inject via `browser.tabs.executeScript` from background page. Ads confirmed blocked on YouTube — video plays without pre-roll/mid-roll ads.
- **Video Speed Controller**: Crashes on `chrome.storage.sync` (not available in Electron). Fixed via polyfill that redirects `sync` → `local`. Also polyfills `chrome.action` → `chrome.browserAction`.
- **Content scripts in isolated world**: `vAPI`/`chrome`/`browser` not available in page main world — expected Chrome extension behavior. The `initContentScriptInjection()` function hooks `web-contents-created` early and injects content scripts on navigation.

### Architecture of extension support
- `extension-loader.js` now exports: `initContentScriptInjection`, `getExtSession`, `getExtApi`
- `initContentScriptInjection()` must be called BEFORE any `BrowserWindow` is created (done in `app.whenReady()` in `main.js`)
- Polyfills injected into extension pages (background, popup) via `dom-ready` event on `web-contents-created`
- Content script injection: listens for `did-navigate` on all webContents, matches URL against manifest patterns, injects via background page's `browser.tabs.executeScript`
