## Context

Ran Playwright E2E diagnostics on button responsiveness bug from last owl session. Found true root cause + secondary issues. No code changed.

### Root cause chain (verified via Playwright)

1. **Changelog overlay (#changelog-overlay, z=1000) blocks ALL clicks** — covers full viewport (1366x694), `pointer-events: auto`. User cannot click anything behind it. The overlay's own buttons (close/dismiss) work.
2. **Even after overlay dismiss, mode buttons don't fire** — TrustManager requires permission grants (`editor::write`, `inspect::read`, `actions::read`, `data::read`, `workflow::run`) that aren't granted at boot. Only `shell::execute` is granted in app.js. Miniapps self-grants. By design but surprising.
3. **Init() runs clean** — `[init] rsidebar buttons attached: 8/8`, 10/10 modules loaded, no JS errors
4. **F12 DevTools IPC chain works** (uncommitted owl fix)
5. **6 ERR_FILE_NOT_FOUND** asset load errors (minor)
6. **Playwright `click()` times out on buttons** — SVG child icons block Playwright's actionability check. Use `{ force: true }` or `dispatchEvent`. Not a user-facing bug.
